### Use cases
```
1. MIDDLEEND along with FRONEND & BACKEND
    - Intermediate layer between client side & microservices.
    - When client send request, we will be able to make decision which microservice should actually respond to this request
    - Message validation
    - Response transformation: response structure for mobile & web browser
    - Rate limiting: If some IP address bombarding requests which needs to be restricted
    - Reverse proxy: routing requests to respective microservices instead of client directly making HTTP requests for microservices with differnt port numbers 
2. API Gateway as MIDDLEEND

```
